### M4. ID/PW 회원가입과 로그인

범위:

- [x] ID와 비밀번호를 입력하는 회원가입 화면
- [x] ID와 비밀번호를 입력하는 로그인 화면
- [x] ID 중복 확인
- [x] 로그인 상태 유지
- [x] 로그아웃
- [x] 로그인한 사용자별 러닝 기록 분리
- [x] 인증 실패·중복 ID·입력 오류 상태 처리
- [x] 회원 탈퇴

MVP 제외:

- [ ] 비밀번호 찾기·재설정
- [ ] 이메일·휴대폰 인증
- [ ] 소셜 로그인
- [ ] 관리자 화면

최소 보안 기준:

- 비밀번호는 평문으로 저장하거나 로그에 출력하지 않는다.
- 비밀번호 해시는 Supabase Auth 또는 검증된 인증 서비스를 사용해 처리한다.
- 사용자 ID는 중복될 수 없으며 로그인 오류에서 다른 사용자의 존재 여부를 과도하게 노출하지 않는다.
- 인증 토큰과 세션 만료를 처리하고, 로그아웃 후 개인 기록을 화면에 남기지 않는다.
- 개인정보 수집을 최소화하더라도 서비스에 저장하는 ID와 러닝 기록의 삭제 방법은 제공한다.

선택 이유와 주의점:

- ID/PW 방식은 사용자에게 익숙하고 사용자별 기록을 분리하기 쉽다.
- 비밀번호 찾기를 제외하면 구현 범위는 줄지만, 비밀번호를 잊은 사용자는 계정을 복구할 수 없다.
- 직접 비밀번호 저장 로직을 만드는 것보다 Supabase Auth 같은 검증된 인증 서비스를 사용하는 편이 안전하다.
- 현재 1차 구현은 화면과 입력 검증만 제공하며, 비밀번호를 로컬에 저장하지 않는다.
- Supabase Auth는 기본적으로 email/password 인증을 지원하므로, MVP에서는 ID를 내부 auth email로 변환한다.
- 실제 회원가입, 로그인, 세션 유지는 Supabase 환경변수 입력 후 동작한다.
- Supabase 인증 실패 응답은 사용자 친화적인 한국어 메시지로 변환한다.
- ID 중복 확인과 회원 탈퇴는 FastAPI가 Supabase Admin API를 호출한다.
- `SUPABASE_SERVICE_ROLE_KEY`는 반드시 `apps/api/.env`에만 저장하고 프론트엔드에 노출하지 않는다.
- 비로그인 사용자가 기록 탭을 누르면 마이 페이지의 로그인 화면으로 이동한다.
- 로컬 MVP 기록 상세에서 저장된 기록을 삭제할 수 있다.
- 로그인 전 임시 기록과 로그인 사용자 기록은 서로 다른 `localStorage` 키로 분리한다.

완료 조건:

- 사용자가 ID/PW로 가입하고 로그인·로그아웃할 수 있다.
- 로그인한 사용자는 자신의 러닝 기록만 조회할 수 있다.
- 비밀번호와 인증 토큰이 로그나 GitHub에 노출되지 않는다.
- 비밀번호 찾기를 지원하지 않는다는 점을 회원가입과 로그인 화면에서 명확히 안내한다.

### M5. 모바일 앱 검증

목표:

- 핵심 러닝 MVP가 갖춰진 뒤 Capacitor로 iPhone 실기기 앱 경험을 검증한다.
- React/Vite 웹앱을 Capacitor로 감싸 iPhone에서 실제 앱처럼 실행해본다.
- 앱 환경에서 카카오맵, 현재 위치, 시설 조회, 러닝 추적·기록 흐름이 유지되는지 검증한다.
- 웹 브라우저와 Capacitor WebView의 차이를 문서화한다.

도입 시점:

- 마커 클러스터링과 러닝 시작·일시정지·종료 흐름이 안정화된 뒤 진행한다. 클러스터링은 완료했다.
- 거리와 시간 계산, 러닝 결과 저장 초안이 있어야 아이폰 실기기 테스트의 의미가 커진다.
- 지금은 즉시 구현하지 않고 API base URL과 Kakao SDK origin 리스크만 기록해둔다.

범위:

- [ ] 핵심 러닝 MVP 이후 `codex/capacitor-ios-preview` 브랜치 생성
- [ ] Capacitor 기본 패키지 설치와 설정 파일 추가
- [ ] iOS 플랫폼 추가
- [ ] `apps/web/dist` 빌드 결과를 Capacitor `webDir`로 연결
- [ ] API 호출을 Capacitor 환경에서도 동작하도록 `VITE_API_BASE_URL` 기반으로 정리
- [ ] 카카오맵 JavaScript SDK가 iOS WebView origin에서 동작하는지 확인
- [ ] iPhone 실기기에서 위치 권한, 현재 위치 마커, 시설 마커 표시 확인

주의점:

- Capacitor 앱 안에서는 Vite dev proxy가 없으므로 `/api` 상대 경로만으로 FastAPI에 접근할 수 없다.
- 카카오맵 SDK는 허용 도메인/origin 설정의 영향을 받으므로 iOS WebView에서 별도 검증이 필요하다.
- 실기기 실행에는 Xcode 설정, 개발자 계정 로그인, iPhone 신뢰 설정이 필요할 수 있다.
- 백그라운드 위치 추적은 MVP 범위에서 바로 확정하지 않고 화면이 켜진 상태의 추적부터 검증한다.

완료 조건:

- 웹 MVP에서 러닝 시작·종료·결과 저장 흐름을 먼저 확인했다.
- Xcode 또는 `npx cap run ios`로 iPhone에서 앱을 실행할 수 있다.
- 앱에서 지도와 현재 위치 권한 흐름을 확인할 수 있다.
- 앱에서 러닝 시작·일시정지·종료 흐름을 확인할 수 있다.
- FastAPI 시설 API 연결 전략을 로컬 개발용과 배포용으로 나눠 문서화한다.
- Capacitor 도입 PR에서 테스트/lint와 민감정보 미포함을 확인한다.
