# 마이 페이지

## 목적

사용자 정보, 앱 설정, 로컬 러닝 기록의 핵심 지표를 확인하게 한다.

## 주요 정보

- 로컬 프로필
- 러닝 대시보드
- 언어 설정
- 지도 제공자 설정
- 위치 권한 안내

## 범위 판단

지도와 러닝 기록 흐름을 먼저 검증한다. 현재 방향은 내 iPhone에서 동작하는 local-first 프로토타입이므로, 마이 페이지는 로그인보다 사용자 정보와 앱 상태를 보여주는 곳으로 축소한다. 인증은 여러 기기 동기화나 공개 서비스 운영이 필요해지는 시점에 다시 검토한다.

## 현재 상태

- 마이 페이지 화면 초안을 제공한다.
- 마이 페이지는 지도 위 카드가 아니라 하나의 독립 대시보드 페이지로 표시한다.
- iPhone safe area와 하단 내비게이션 뒤까지 배경을 덮어 지도 화면이 비치지 않게 한다.
- 위치 권한과 앱 상태 안내를 제공한다.
- 이전 실험으로 ID/PW 로그인, 회원가입, Supabase Auth 연결 흐름을 구현해봤다.
- 현재 MVP에서는 로그인 기능을 핵심 범위에서 내리고, 로컬 프로필과 앱 설정 중심으로 재정리했다.
- 로컬 기록 개수, 누적 거리, 최근 기록 날짜를 요약해 보여준다.
- 총 거리, 총 러닝 횟수, 월간 거리, 최장 거리, 평균 페이스를 대시보드 카드로 보여준다.
- 주간·월간 목표를 사용자가 직접 설정하고 목표 대비 진행률을 확인할 수 있다.
- D3 기반 최근 4개월 거리 그래프, 목표 대비 비교 그래프, 이번 달 러닝 날짜 달력을 제공한다.
- 설정 카드의 지도 전략은 Apple MapKit native 적용 상태, 시설 데이터는 local-first JSON 1차 완료 상태로 표시한다.
- 비밀번호 찾기, 이메일 인증, 소셜 로그인, 회원 탈퇴는 추후 확장 범위로 분리한다.

## 대시보드 구현

마이 페이지는 local-first MVP에서 계정 관리 화면보다 러닝 상태를 이해하는 화면으로 더 가치가 크다. 따라서 로그인 기능은 줄이고, 내 기기에 저장된 기록을 바탕으로 대시보드를 먼저 구현한다.

- 총 뛴 거리
- 총 러닝 횟수
- 최근 러닝 날짜와 거리
- 월간 거리
- 최장 러닝 거리
- 평균 페이스
- 목표 설정
- 이번 주/이번 달 목표 대비 진행률
- D3 기반 최근 4개월 거리 그래프
- D3 기반 목표 대비 비교 그래프
- 달력 기반 러닝 날짜 표시

## 추가 후보

아래 기능은 현재 대시보드의 다음 고도화 후보로 둔다.

- 대시보드 카드 순서 사용자 설정
- 주간·월간·전체 기간 전환
- 그래프 없는 간단 모드
- 평균 페이스 추이 그래프
- 목표 대비 비교 그래프 고도화
- 러닝 기록 리셋

## 차트 라이브러리 판단

대시보드 차트는 `three.js`보다 `D3`를 우선 사용한다. 현재 필요한 시각화는 3D 연출이 아니라 거리, 목표, 달력처럼 작은 데이터를 정확하게 읽히게 하는 2D SVG 차트이기 때문이다. `three.js`는 WebGL 기반의 3D 지도나 몰입형 시각화가 필요해질 때 다시 검토한다.

## 완료 조건

- 로그인하지 않아도 현재 사용자 정보와 앱 상태를 이해할 수 있다.
- 로컬 러닝 기록 요약과 설정 진입점이 명확하다.
- 위치 권한 문제를 해결할 수 있는 안내가 있다.
- 지도 화면과 분리된 단일 페이지로 보여서 대시보드에 집중할 수 있다.
