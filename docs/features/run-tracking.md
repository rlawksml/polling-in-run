# 러닝 추적

## 해결하려는 문제

사용자가 러닝을 시작한 뒤 이동 거리와 시간을 기록해야 한다.

## 구현 방향

브라우저 Geolocation API의 `watchPosition()`으로 위치 변화를 받고, 좌표 간 거리를 계산해 경로와 누적 거리를 만든다.

## 장점

- 별도 네이티브 앱 없이 웹 기술로 MVP를 검증할 수 있다.
- 위치 권한, 정확도, 상태 모델을 프론트엔드에서 학습할 수 있다.

## 단점과 위험

- 화면이 꺼지거나 브라우저가 백그라운드로 이동하면 추적이 중단될 수 있다.
- GPS 오차로 실제보다 긴 경로가 기록될 수 있다.
- 지속적인 고정밀 위치 추적은 배터리를 많이 사용한다.

## 초기 검증 범위

- 화면이 켜진 상태에서만 경로 추적
- 위치 정확도가 낮은 좌표 필터링
- 시작·일시정지·종료 상태 전환
- 시간과 누적 거리 계산

## 재검토 조건

NRC나 Adidas Running 수준의 백그라운드 추적이 필수라면 React Native 또는 네이티브 앱 전환을 검토한다.

## 완료 조건

- 테스트 가능한 상태 전환 모델이 있다.
- 위치 추적을 종료하면 `clearWatch()`가 호출된다.
- GPS 오류와 모바일 웹 한계를 사용자에게 설명한다.
