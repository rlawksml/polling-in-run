# 러닝 추적

## 해결하려는 문제

사용자가 러닝을 시작한 뒤 이동 거리와 시간을 기록해야 한다.

## 구현 방향

브라우저 Geolocation API의 `watchPosition()`으로 위치 변화를 받고, 좌표 간 거리를 계산해 경로와 누적 거리를 만든다.

## 장점

- 별도 네이티브 앱 없이 웹 기술로 MVP를 검증할 수 있다.
- 위치 권한, 정확도, 상태 모델을 프론트엔드에서 학습할 수 있다.

## 단점과 위험

- 웹 브라우저만 사용할 때는 화면이 꺼지거나 브라우저가 백그라운드로 이동하면 추적이 중단될 수 있다.
- Capacitor iOS 앱에서는 native background location 브릿지를 통해 WebView 대신 `CLLocationManager`가 좌표를 수집하는 1차 구현을 추가했다.
- 단, iOS 실기기에서 위치 권한을 `항상 허용`으로 승인하고 화면 잠금·앱 전환 상태를 검증해야 NRC/Adidas Running에 가까운 안정성을 판단할 수 있다.
- GPS 오차로 실제보다 긴 경로가 기록될 수 있다.
- 지속적인 고정밀 위치 추적은 배터리를 많이 사용한다.

## 초기 검증 범위

- 시작·일시정지·종료 상태 전환. 1차 완료
- 시간과 누적 거리 표시 뼈대. 1차 완료
- 화면이 켜진 상태에서만 경로 추적. 1차 완료
- 위치 정확도가 낮은 좌표 필터링. 1차 완료
- 시간과 누적 거리 계산. 1차 완료
- 모바일 브라우저에서 러닝 시작, 저장, 기록 조회 흐름 확인. 실기기 확인 대기

## 구현 기록

### 2026-06-17

- `idle → running → paused → finished` 상태 모델을 추가했다.
- 홈 화면의 러닝 시작 버튼을 러닝 진행 화면으로 연결했다.
- 러닝 진행 화면에서 시간, 거리, 평균 페이스, 일시정지, 재개, 종료를 표시한다.
- 1차 버전의 거리는 `0.00 km`로 두고, GPS 기반 거리 계산은 다음 단계로 분리했다.

### 2026-06-21

- 러닝 중 `watchPosition()`으로 위치 포인트를 수집한다.
- 일시정지와 종료 시 `clearWatch()`로 실시간 위치 추적을 해제한다.
- 정확도 80m를 초과하는 좌표는 거리 계산 후보에서 제외하고 사용자에게 안내한다.
- 브라우저가 실시간 위치 추적을 지원하지 않거나 오류가 발생하면 러닝 패널에 상태를 표시한다.
- 좌표 간 haversine 거리 계산으로 누적 거리와 평균 페이스를 표시한다.
- 모바일 브라우저에서 확인할 항목을 [`모바일 실기기 검증`](../qa/mobile-real-device-check.md) 체크리스트로 분리했다.

### 2026-06-24

- 현재 러닝 추적은 앱이 foreground에 있고 화면이 켜진 상태를 우선 지원 범위로 둔다.
- iOS native background location 브릿지를 추가했다.
- `BackgroundLocationPlugin`은 `CLLocationManager`를 사용하고, `UIBackgroundModes: location`과 항상 위치 권한 문구를 `Info.plist`에 추가했다.
- React 러닝 훅은 Capacitor native 환경에서 브라우저 `navigator.geolocation`보다 native plugin 이벤트를 우선 사용한다.

### 2026-06-30

- 실제 이동 경로가 직선처럼 단순화되는 문제를 줄이기 위해 현재 위치 보조 샘플링을 3초에서 1초 간격으로 조정했다.
- 경로 포인트 저장 기준을 2m에서 0.75m로 낮춰 코너와 짧은 방향 전환 좌표가 더 잘 남도록 했다.
- 다만 iOS WebView가 실제 중간 좌표를 제공하지 않거나 앱이 background로 내려가면 누락된 경로를 앱이 복원할 수는 없다.

### 2026-07-01

- 경로 표시용 포인트 저장 기준과 누적 거리 계산 기준을 분리했다.
- 0.15m 이상 좌표 변화는 경로 모양을 위해 저장하고, 0.75m 이상 이동만 누적 거리에 더한다.
- 짧은 코너와 방향 전환 좌표를 더 잘 보존하되, GPS 미세 흔들림이 거리로 과도하게 더해지는 것을 줄인다.
- Capacitor iOS용 `BackgroundLocation` 브릿지를 추가해 화면 잠금·앱 전환 중에도 native 위치 이벤트를 받을 수 있는 기반을 마련했다.
- 남은 검증은 Xcode 실기기에서 `항상 허용` 권한, 화면 잠금, 다른 앱 전환, 러닝 종료 후 기록 저장까지 확인하는 것이다.

## 재검토 조건

NRC나 Adidas Running 수준의 백그라운드 추적 품질을 목표로 한다면 현재 native bridge를 실기기에서 검증한 뒤, 배터리 소모, 위치 권한 안내, 앱 심사 문구, 장거리 기록 누락 여부를 추가로 점검한다.

## 완료 조건

- 테스트 가능한 상태 전환 모델이 있다.
- 위치 추적을 종료하면 `clearWatch()`가 호출된다.
- GPS 오류와 모바일 웹 한계를 사용자에게 설명한다.
- 거리 계산 전 단계에서 수집한 위치 포인트 수를 확인할 수 있다.
- 수집한 위치 포인트를 기반으로 누적 거리와 평균 페이스를 계산할 수 있다.
- 모바일 브라우저에서 시작, 일시정지, 재개, 종료, 저장, 기록 조회를 확인한다.
