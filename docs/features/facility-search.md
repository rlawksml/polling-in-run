# 주변 편의시설

## 해결하려는 문제

러닝 전후와 러닝 중에 가까운 공공 음수대와 화장실을 빠르게 찾아야 한다.

## 구현 방향

FastAPI가 여러 공공데이터 원본을 공통 시설 모델로 정규화하고, React는 현재 위치 주변의 정규화된 시설만 표시한다.

```text
공공데이터 원본
→ FastAPI 정규화·거리 필터
→ 공통 시설 API
→ React 지도 마커
```

## FastAPI를 두는 이유

### 장점

- 서로 다른 공공데이터 형식을 프론트엔드에서 숨길 수 있다.
- 외부 API 키와 데이터 정제 규칙을 서버에서 관리할 수 있다.
- 데이터 출처가 변경되어도 프론트엔드 계약을 유지하기 쉽다.

### 단점

- 서버 배포와 장애 지점이 추가된다.
- 초기에는 정적 데이터만 표시하는 방식보다 구현량이 많다.

## 공통 시설 모델 초안

- `id`
- `type`: `water` 또는 `restroom`
- `name`
- `latitude`
- `longitude`
- `address`
- `source`
- `updated_at`

## 완료 조건

- 현재 위치 반경 안의 시설을 조회할 수 있다.
- 물과 화장실 마커를 구분할 수 있다.
- 데이터가 없거나 조회에 실패한 상태를 안내한다.
