# Local-first iPhone Prototype

## 배경

Polling In Run은 대규모 공개 서비스보다, 러닝 중 필요한 편의시설 확인과 러닝 기록 흐름을 내 iPhone에서 직접 검증하는 토이프로젝트다.

초기에는 FastAPI, Supabase Auth, PostgreSQL까지 포함한 풀스택 구조를 실험했다. 하지만 현재 목표에서는 서버 운영, 계정 관리, 데이터베이스 동기화보다 모바일 사용 경험을 빠르게 검증하는 것이 더 중요하다.

## 선택 방향

현재 MVP는 local-first iPhone prototype으로 재정의한다.

```text
iPhone App
├─ 지도와 현재 위치
├─ 로컬 시설 데이터
├─ 로컬 러닝 기록
├─ 마이 페이지 사용자 정보·설정
└─ 추후 확장
   ├─ FastAPI 공공데이터 갱신
   └─ Supabase Auth/PostgreSQL 동기화
```

## 유지할 기능

- 현재 위치 기반 지도
- 음수대와 화장실 마커
- 정규화된 시설 JSON 로컬 조회
- 시설 상세 정보
- 외부 지도 앱 길찾기
- 러닝 시작, 일시정지, 종료
- 거리, 시간, 페이스 계산
- 러닝 기록과 메모 로컬 저장
- 기록 조회와 삭제
- 마이 페이지의 사용자 정보와 앱 설정

## 축소할 기능

- ID/PW 회원가입과 로그인
- 비밀번호 찾기
- 이메일 인증
- 소셜 로그인
- 회원 탈퇴
- Supabase PostgreSQL 기록 동기화
- 서버 배포

이 기능들은 공개 서비스나 멀티 디바이스 동기화가 필요해지는 시점에 다시 검토한다.

## 지도 전략

Kakao Maps JavaScript SDK는 Web 플랫폼 도메인 검증을 사용한다. Capacitor 정적 앱은 `capacitor://localhost` origin에서 실행되므로, iPhone 앱 내부에서 SDK가 안정적으로 동작하지 않을 수 있다.

현재 iPhone local-first 앱의 1차 지도 후보는 Apple MapKit native다. 길찾기는 앱 내부 구현보다 외부 지도 앱 연동을 우선한다. 자세한 비교와 결정은 [`지도 전략`](./map-strategy.md)에 정리한다.

## 시설 데이터 방향

FastAPI는 공공데이터 원본을 정규화하는 도구로 유지하되, iPhone 앱 런타임에서는 서버 없이 `apps/web/public/data/facilities.json`을 먼저 읽는다.

```text
서울 열린데이터광장 + 공간데이터마켓 원본
→ FastAPI 정규화 함수
→ npm run export:facilities
→ public/data/facilities.json
→ 앱 내부 조회
```

현재 JSON은 실제 음수대와 화장실 6,477건을 담고 있다. 지도 영역 필터와 현재 위치 거리 계산은 프론트에서 수행한다. 데이터 갱신이 필요할 때만 export 명령을 다시 실행한다.

## 마이 페이지 방향

마이 페이지는 로그인 화면이 아니라 로컬 사용자 정보와 앱 상태를 보여주는 공간으로 둔다.

- 표시 이름 또는 로컬 프로필
- 러닝 기록 요약과 대시보드
- 위치 권한 상태
- 지도 제공자 또는 길찾기 앱 설정
- 데이터 갱신 안내

마이 페이지는 총 거리, 최장 거리, 평균 페이스, 목표 설정, 목표 대비 비교 그래프, 달력 기반 러닝 날짜를 보여주는 1차 대시보드로 확장했다. 목표 입력 UI, 그래프 라벨, empty state, 기록 리셋, 평균 페이스 추이 그래프는 추후 polish 작업으로 둔다. 메모가 있는 러닝만 모아보기는 기록 페이지 필터로 분리했다.

## 확장 조건

다음 조건이 생기면 서버와 인증을 다시 도입한다.

- 여러 기기에서 기록을 공유해야 한다.
- 다른 사용자에게 서비스를 공개한다.
- 시설 데이터를 앱 업데이트 없이 자동 갱신해야 한다.
- 사진 업로드나 클라우드 백업이 필요하다.

## 결정 요약

현재는 기능을 더 크게 벌리기보다 내 iPhone에서 안정적으로 동작하는 러닝 편의 앱을 만든다. 서버와 인증은 포기한 것이 아니라, local-first MVP 이후의 확장 경로로 남긴다.
