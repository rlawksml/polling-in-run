# 카카오맵과 현재 위치

## 해결하려는 문제

한국의 러너가 현재 위치와 주변 공공시설을 빠르게 확인해야 한다.

## 선택

MVP 지도 제공자로 Kakao Maps JavaScript API를 사용한다. 현재 위치는 브라우저 Geolocation API로 가져온다.

## 카카오맵을 선택한 이유

### 장점

- 국내 지도, 주소, 장소 검색에 적합하다.
- 한국 사용자에게 익숙한 지도 표현을 제공한다.
- MVP 규모에서 무료 쿼터가 충분하다.
- 장소 검색과 주소 변환 기능을 함께 검토할 수 있다.

### 단점

- 해외 확장에는 Google Maps보다 불리하다.
- 지도 스타일을 제품 브랜드에 맞게 깊게 변경하기 어렵다.
- JavaScript SDK가 전역 `kakao` 객체를 사용하므로 React 생명주기와 타입을 직접 관리해야 한다.

### 대안

Google Maps는 해외 확장, 지도 스타일링, 다양한 지도 제품이 필요할 때 더 적합하다. 현재는 국내 공공시설 탐색과 빠른 MVP 완성을 우선하므로 선택하지 않는다.

## API 키 관리

- `apps/web/.env.local`: 실제 Kakao JavaScript 키. GitHub 커밋 금지.
- `apps/web/.env.example`: 필요한 환경변수 이름만 공유.
- Kakao Developers JavaScript SDK 도메인에 `http://localhost:5173` 등록.
- 향후 REST API 키와 관리자 권한 키는 FastAPI 환경변수에서만 사용.

## 위치 추적 경계

- 지도 API는 지도를 그린다.
- Geolocation API는 사용자의 위치를 제공한다.
- FastAPI는 시설 데이터와 러닝 기록을 처리한다.

## 주요 상태

- SDK 로딩 중·실패
- 위치 권한 요청 중
- 권한 허용
- 권한 거부
- 브라우저 미지원
- 위치 조회 시간 초과
- 정확도 낮음

## 완료 조건

- 홈 화면에서 카카오맵이 렌더링된다.
- 사용자 현재 위치와 정확도 범위를 표시한다.
- 위치 권한 거부와 오류 상태에 재시도 방법이 있다.
- 키가 없어도 개발자가 원인을 이해할 수 있는 안내 화면이 보인다.

## 구현 기록

### 2026-06-14

- Kakao Maps SDK를 중복 없이 불러오는 로더를 구현했다.
- 지도 객체와 React 화면 상태를 분리했다.
- 현재 위치의 로딩, 성공, 권한 거부, 시간 초과, 미지원 상태를 구현했다.
- 위치 권한 거부 상태와 지도 SDK 실패 상태를 모바일 화면에서 확인했다.
- 초기 SDK 요청은 Kakao Developers 웹 도메인 미등록으로 `domain mismatched` 응답을 받았다.
- `http://localhost:5173`과 `http://127.0.0.1:5173`을 웹 플랫폼 도메인에 등록한 뒤 SDK 응답과 실제 모바일 지도 렌더링을 확인했다.
