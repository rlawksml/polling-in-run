# 모바일 실기기 검증

## 목적

모바일 브라우저에서 Polling In Run의 핵심 흐름이 실제 사용 가능한지 확인한다.

검증 범위는 두 가지다.

- M1: 카카오맵, 현재 위치, 주변 시설 조회
- M3: 러닝 시작, 위치 추적, 결과 저장, 기록 조회

이 문서는 모바일 브라우저와 iPhone 앱 실기기 검증 결과를 기록한다. 2026-06-24 사용자가 iPhone에서 핵심 지도 흐름을 확인했고, 결과를 `TODO.md`에 반영했다.

## 준비 조건

- PC와 스마트폰이 같은 Wi-Fi에 연결되어 있어야 한다.
- Web dev server는 네트워크 접속을 허용하는 `--host 0.0.0.0` 옵션으로 실행한다.
- FastAPI dev server가 켜져 있어야 한다.
- 스마트폰에서는 PC의 내부 IP 주소로 접속한다. 현재 개발 환경 예시는 `http://192.168.45.3:5173/`이다.
- Kakao Developers Web 플랫폼 도메인에 스마트폰에서 접속할 URL을 임시 등록한다.
  - 예: `http://192.168.45.3:5173`
  - 내부 IP는 네트워크가 바뀌면 달라질 수 있으므로, 바뀔 때마다 재등록이 필요하다.
- `apps/web/.env.local`과 `apps/api/.env`의 실제 키는 Git에 커밋하지 않는다.

## M1 체크리스트: 지도와 현재 위치

- [ ] 스마트폰 브라우저에서 앱 URL에 접속할 수 있다.
- [ ] 카카오맵이 `domain mismatched` 없이 렌더링된다.
- [ ] 위치 권한 요청이 표시된다.
- [x] 위치 권한을 허용하면 현재 위치 마커가 표시된다.
- [ ] 현재 위치로 이동 버튼이 동작한다.
- [ ] 확대/축소 버튼이 동작한다.
- [ ] 지도 이동 후 현재 지도 영역 기준으로 시설 요청이 다시 실행된다.
- [ ] 음수대와 화장실 마커 또는 클러스터가 표시된다.
- [ ] 시설을 선택하면 이름, 주소, 거리 정보가 표시된다.
- [ ] 카카오맵 길찾기 버튼이 외부 카카오맵 경로로 연결된다.

## M3 체크리스트: 러닝과 기록

- [ ] 홈 화면에서 러닝 시작 버튼을 누르면 러닝 화면으로 이동한다.
- [ ] 러닝 시작 후 위치 권한 상태가 정상적으로 유지된다.
- [ ] 화면이 켜진 상태에서 이동하면 위치 포인트가 수집된다.
- [ ] 이동 거리에 따라 거리와 평균 페이스가 갱신된다.
- [ ] 일시정지를 누르면 시간과 위치 추적이 멈춘다.
- [ ] 재개를 누르면 러닝 추적이 다시 이어진다.
- [ ] 종료를 누르면 러닝 결과 화면으로 이동한다.
- [ ] 결과 화면에서 거리, 시간, 평균 페이스를 확인할 수 있다.
- [ ] 메모를 입력하고 기록을 저장할 수 있다.
- [ ] 기록 탭에서 저장한 기록 목록을 확인할 수 있다.
- [ ] 기록 상세에서 저장한 메모와 러닝 정보를 확인할 수 있다.

## 주의점

- 모바일 브라우저는 화면이 꺼지거나 앱이 백그라운드로 이동하면 위치 추적이 제한될 수 있다.
- 로컬 개발 URL은 배포 URL과 다르므로 Kakao SDK 도메인 등록을 분리해서 관리해야 한다.
- 현재 러닝 기록은 `localStorage`에 저장되므로 기기와 브라우저가 바뀌면 기록이 공유되지 않는다.
- 실제 사용자 계정별 기록 공유는 Supabase Auth와 DB 연동 이후 검증한다.
- 앱처럼 안정적인 백그라운드 위치 추적이 필요하면 Capacitor 또는 네이티브 앱 검토가 필요하다.

## 완료 판단

M1의 `모바일 브라우저에서 실제 위치 확인`은 지도 렌더링, 현재 위치 마커, 주변 시설 표시가 iPhone 실기기에서 확인되어 완료로 본다.

M3의 러닝 흐름은 시작, 일시정지, 재개, 종료, 저장, 기록 조회가 같은 모바일 브라우저에서 한 번 이상 이어지면 완료로 본다.
