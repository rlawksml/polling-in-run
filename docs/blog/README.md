# Blog Drafts

Polling In Run을 개발하며 블로그나 포트폴리오 회고로 확장할 글감을 모아두는 폴더다.

## 작성 원칙

- 단순 작업 기록보다 문제, 선택지, 판단 이유, 결과를 중심으로 쓴다.
- 독자가 프론트엔드 개발자라고 가정하고 백엔드와 공공데이터 맥락을 풀어서 설명한다.
- 실제 구현 파일과 문서 링크를 남겨 나중에 검증 가능한 글로 다듬는다.
- 비용, 보안, 운영 제약처럼 불편하지만 중요한 판단도 숨기지 않는다.
